# Momenta - 运动打卡程序 - 需求研究

> 在动手之前，先搞清楚要做什么、为什么做、有没有人做过。

## 目标

### 我要解决什么问题？
用户每周进行4次健身和篮球等运动，希望有一个简单、方便的工具来：
1. 记录每次运动（快速打卡，无需详细记录训练内容）
2. 在应该运动的时候收到提醒（避免忘记）
3. 通过数据可视化和激励机制保持运动动力

### 这个问题真的值得解决吗？
是的。现有运动App要么功能过于复杂（需要记录详细训练数据），要么缺少有效的激励机制。用户的核心诉求是"简单打卡 + 持续激励"，这是一个明确且合理的痛点。

### 我希望做出来的东西是什么样的？
一个移动端优先的PWA（渐进式Web应用），打开浏览器即可使用，无需下载安装。核心体验是"点两下完成打卡"——选择运动类型，确认，完成。同时提供Streak连续打卡、成就系统、数据可视化和目标追踪等激励功能。

### 目标用户是谁？
个人用户，主要为自己使用。不需要多用户支持或社交功能。

### 核心使用场景是什么？
1. **运动后快速打卡**：健身/篮球结束后，打开手机浏览器，选择运动类型，点确认，2秒内完成记录
2. **查看运动历史**：想看看这周/这个月运动了几次，打开App一目了然
3. **接收运动提醒**：在设定的运动时间收到浏览器推送通知，避免遗忘
4. **查看激励数据**：Streak天数、成就徽章、运动统计图表，获得持续动力

## 市场调研

### 现有产品分析

#### 产品1：Keep
- **链接**：https://keep.com
- **优点**：
  - 国内用户基数大，社区活跃
  - 提供丰富的训练课程和视频指导
  - 有打卡和记录功能
  - 运动数据分析较为完善
- **缺点**：
  - 功能过于复杂，包含商城、智能硬件、社交等大量非核心功能
  - 需要下载安装App，占用手机存储
  - 打卡流程不够极简，需要选择课程、记录详细数据
  - 广告和推广内容较多，干扰用户体验
  - 对个人简单打卡场景来说太重

#### 产品2：Strava
- **链接**：https://www.strava.com
- **优点**：
  - 国际知名运动社交平台，支持多种运动类型
  - GPS轨迹记录和数据分析非常专业
  - 有挑战和成就系统
  - 社区激励氛围好
- **缺点**：
  - 核心依赖GPS追踪，不适合健身房等室内运动
  - 社交属性强，需要注册账号、添加好友
  - 界面复杂，学习成本较高
  - 需要下载App，对个人简单打卡场景过重
  - 部分高级功能需要订阅付费

#### 产品3：Apple Fitness / Google Fit（系统自带）
- **优点**：系统集成，无需额外安装
- **缺点**：
  - 依赖设备传感器，手动记录体验差
  - 激励机制薄弱
  - 自定义程度低

### 差异化定位

**Momenta 的核心差异：**
1. **极简打卡体验**：无需下载，浏览器打开即用，2秒完成打卡
2. **个人专注**：无社交、无社区、无广告，纯粹的个人运动记录工具
3. **组合激励机制**：Streak + 成就 + 可视化 + 目标追踪，全方位激励
4. **智能提醒**：固定时间 + 基于打卡记录的智能提醒，双重保障不遗忘
5. **PWA技术栈**：不需要App Store审核，跨平台（iOS/Android/桌面），自动更新

**核心优势：**
- 使用门槛极低（有浏览器就行）
- 打卡流程极简（选择运动类型 → 确认 → 完成）
- 激励机制完整且即时反馈
- 数据完全本地存储，隐私安全

## 核心需求（按优先级排序）

### P0 - MVP必须实现
1. **运动打卡**：选择预设运动类型（健身、篮球、跑步、游泳、瑜伽等），一键完成打卡，可添加备注
2. **打卡历史**：按时间线展示所有打卡记录，支持查看和删除
3. **Streak连续打卡**：显示当前连续运动天数，断了清零
4. **固定时间提醒**：设置每周运动时间，到点推送浏览器通知
5. **基础数据展示**：显示本周/本月运动次数

### P1 - 重要但可延后
6. **成就系统**：解锁徽章（首周全勤、月运动20次、篮球达人等）
7. **数据可视化**：图表展示运动频率趋势
8. **目标追踪**：设置每周/每月运动目标，显示完成进度
9. **智能提醒**：基于上次打卡时间计算，超期未运动时主动提醒
10. **自定义运动类型**：用户可添加自己的运动类型

### P2 - nice to have
11. 导出运动数据
12. 深色模式
13. 数据备份/恢复

## 非核心需求（明确不做）

1. **多用户/社交功能**：不登录、不加好友、不分享
2. **详细训练记录**：不记录组数、重量、动作细节
3. **GPS轨迹追踪**：不做路线记录
4. **视频课程/训练计划**：不内置教学内容
5. **商城/电商**：不卖任何东西
6. **云同步**：数据本地存储（P1可考虑简单的导出备份）

## 关键约束

### 技术约束
- **必须支持PWA**：Service Worker、Web App Manifest、离线访问
- **浏览器推送通知**：需要处理权限申请和通知管理
- **移动端优先**：UI/UX必须以手机体验为优先
- **本地数据存储**：使用IndexedDB或localStorage，无后端服务器

### 时间约束
- MVP目标：1-2周完成核心功能
- 完整版本：1个月内完成P0+P1功能

### 资源约束
- 纯前端项目，无后端运维成本
- 免费部署（GitHub Pages / Vercel / Netlify）

## 结论和下一步

### 调研结论
市场上缺少一个**极简、个人化、无需安装**的运动打卡工具。Keep和Strava功能强大但过于复杂，不适合"只想简单记录和获得激励"的用户。Momenta的定位是填补这个空白——做运动打卡领域的"极简主义"产品。

### 是否继续推进项目？
✅ **是**。需求明确，差异化定位清晰，技术可行性强。

### 项目方向是否需要调整？
无需调整。核心方向（PWA + 极简打卡 + 组合激励）符合用户需求和市场空白。
