# Momenta - 产品需求文档 (PRD)

> 产品需求文档是整个流程中最重要的一步。好的PRD能让AI准确理解你的意图，生成高质量的代码。

## 产品概述

Momenta 是一款面向个人运动爱好者的极简运动打卡PWA（渐进式Web应用）。用户无需下载安装，通过手机浏览器打开即可使用。核心体验是"2秒完成打卡"——选择运动类型，确认，完成。应用通过Streak连续打卡、成就系统、数据可视化和目标追踪等组合激励机制，帮助用户建立并维持运动习惯。数据完全本地存储，无需注册登录，保护用户隐私。

## 目标用户

**用户画像**：
- 年龄：20-40岁
- 运动习惯：每周规律运动（如健身4次+篮球等），有明确的运动计划
- 技术背景：普通手机用户，不需要专业运动数据
- 核心痛点：容易忘记打卡、缺少运动动力反馈、现有App太复杂
- 使用场景：运动结束后快速记录、查看运动统计、接收运动提醒

## 核心功能

### 必须做（MVP - P0）

#### 功能1：运动快速打卡

**描述**：用户在运动结束后，通过极简流程完成一次打卡记录。

- **输入**：
  - 运动类型（从预设列表选择：健身、篮球、跑步、游泳、瑜伽、骑行、羽毛球、乒乓球、其他）
  - 可选备注（文本，最多100字）
  - 打卡时间（自动取当前时间，用户可手动修改为过去24小时内的时间）

- **输出**：
  - 打卡成功确认（显示打卡的运动类型和时间）
  - 更新后的Streak天数（如适用）
  - 今日已完成运动提示

- **规则**：
  - 同一天可以多次打卡不同运动类型
  - 同一运动类型同一天可多次打卡（如早晚各跑一次）
  - 打卡时间不能为未来时间
  - 打卡时间不能早于最早一条记录的时间（避免数据混乱）

- **边界情况**：
  - 用户未选择运动类型：提示"请选择运动类型"
  - 用户设置的未来时间：提示"不能记录未来的运动"
  - 用户离线时打卡：本地保存，联网后同步（如实现云同步）/ 仅本地保存（当前版本）

- **验收标准**：
  - [ ] 从打开App到完成打卡平均不超过5秒（不包括选择时间）
  - [ ] 预设运动类型至少包含8种
  - [ ] 备注字段可选，不填不影响打卡
  - [ ] 打卡后页面显示成功动画/提示

#### 功能2：打卡历史记录

**描述**：以时间线形式展示所有打卡记录，支持查看和删除。

- **输入**：
  - 用户查看历史列表（无输入）
  - 用户删除某条记录（点击删除按钮并确认）
  - 用户筛选查看（按运动类型、按日期范围）

- **输出**：
  - 按时间倒序排列的打卡记录列表
  - 每条记录显示：运动类型图标、运动类型名称、打卡时间、备注（如有）
  - 支持分组显示（如"今天"、"昨天"、"本周"、"更早"）

- **规则**：
  - 记录按打卡时间倒序排列（最新的在最上面）
  - 删除记录需要二次确认（防止误删）
  - 删除记录后Streak需要重新计算

- **边界情况**：
  - 无记录时：显示空状态提示"还没有打卡记录，快去运动吧！"
  - 大量记录时：支持虚拟滚动或分页加载

- **验收标准**：
  - [ ] 记录列表加载时间不超过1秒（100条记录以内）
  - [ ] 删除记录后列表即时更新
  - [ ] 支持按运动类型筛选
  - [ ] 空状态有友好的引导提示

#### 功能3：Streak连续打卡

**描述**：计算并展示用户连续运动的天数，激励用户保持连续性。

- **输入**：
  - 系统自动根据打卡记录计算
  - 用户手动修正（允许用户手动设置/修正Streak，应对误删记录等情况）

- **输出**：
  - 当前Streak天数（显示在首页显眼位置）
  - 历史最高Streak记录
  - Streak状态（进行中/已中断）

- **规则**：
  - Streak定义：连续N天每天至少有一次打卡（不要求运动类型相同）
  - 今天有打卡 → Streak +1（如果是连续的第N天）
  - 昨天有打卡但今天没有 → Streak保持（今天还没过完，允许当天打卡）
  - 前天有打卡但昨天和今天都没有 → Streak中断，重置为0
  - 允许用户设置"休息日"（如每周固定1天休息，不计入中断）

- **边界情况**：
  - 首次使用：Streak为0
  - 删除记录导致Streak变化：重新计算并更新显示
  - 跨时区旅行：以用户本地时间为准

- **验收标准**：
  - [ ] Streak计算准确（测试用例：连续打卡3天后中断1天，应显示Streak=0）
  - [ ] 首页醒目显示当前Streak天数
  - [ ] 支持设置每周固定休息日（至少1天）
  - [ ] 中断Streak时有视觉提示（如火焰图标熄灭）

#### 功能4：固定时间浏览器推送提醒

**描述**：在设定的运动时间，通过浏览器推送通知提醒用户该去运动了。

- **输入**：
  - 用户设置提醒时间（如每周一/三/五/日晚上19:00）
  - 用户设置提醒内容（可选，默认"该去运动啦！加油！"）
  - 用户开启/关闭推送权限

- **输出**：
  - 浏览器推送通知（显示运动类型建议和激励文案）
  - 通知设置保存状态

- **规则**：
  - 提醒基于Service Worker和Push API实现
  - 提醒时间精确到分钟（如19:00）
  - 支持设置多个提醒（如周一健身、周三篮球、周五健身）
  - 如果当天已经打过卡，不发送提醒（可选设置）
  - 用户未授予通知权限时，友好提示如何开启

- **边界情况**：
  - 用户未授予通知权限：显示提示引导用户开启
  - 用户设备不支持推送：降级为App内提示（打开App时显示）
  - 用户离线：提醒在设备端排队，联网后发送（或仅本地提醒）

- **验收标准**：
  - [ ] 成功申请通知权限并保存用户选择
  - [ ] 在设定时间准时推送通知（误差不超过1分钟）
  - [ ] 支持设置至少3个不同时间段的提醒
  - [ ] 已打卡当天不再提醒（如用户开启此选项）
  - [ ] 提供关闭提醒的开关

#### 功能5：基础数据统计

**描述**：在首页展示基础运动统计数据，让用户快速了解自己的运动情况。

- **输入**：
  - 系统自动从打卡记录统计

- **输出**：
  - 本周运动次数（本周一至今日）
  - 本月运动次数（本月1日至今日）
  - 总运动次数（历史累计）
  - 本周/本月各运动类型分布（饼图或列表）

- **规则**：
  - 统计基于本地存储的打卡记录
  - 周统计以自然周为准（周一至周日）
  - 月统计以自然月为准

- **边界情况**：
  - 无记录时：所有统计显示为0，有鼓励文案
  - 跨周/跨月：自动切换统计周期

- **验收标准**：
  - [ ] 统计数据与打卡记录完全一致
  - [ ] 首页加载时统计数据即时显示
  - [ ] 周/月统计周期正确（周一为周起始）

### 重要但可延后（P1）

#### 功能6：成就系统

**描述**：用户达成特定目标后解锁成就徽章，获得即时成就感。

- **预设成就**：
  - 🏃‍♂️ **初次启航**：完成首次打卡
  - 🔥 **三日燃**：连续打卡3天
  - ⚡ **一周全勤**：连续打卡7天
  - 🏆 **月坚守**：一个月内打卡20次
  - 🏀 **篮球达人**：累计打篮球10次
  - 💪 **健身狂人**：累计健身30次
  - 🌟 **百变运动**：体验过5种不同运动类型
  - 🎯 **目标达成**：完成一次设定的周目标

- **规则**：
  - 成就状态：未解锁（灰色）/ 已解锁（彩色，带解锁时间）
  - 解锁时显示成就弹窗动画
  - 成就进度实时计算（如"篮球达人 7/10"）

- **验收标准**：
  - [ ] 至少8个预设成就
  - [ ] 成就解锁时有明显的视觉反馈
  - [ ] 成就页面展示解锁进度

#### 功能7：数据可视化

**描述**：通过图表直观展示运动趋势和分布。

- **图表类型**：
  - 周/月运动频率柱状图（X轴：日期，Y轴：运动次数）
  - 运动类型分布饼图/环形图
  - Streak趋势折线图（最近30天）

- **规则**：
  - 图表支持时间范围切换（本周/本月/最近30天/全部）
  - 空数据时显示占位提示

- **验收标准**：
  - [ ] 图表响应式，适配手机屏幕
  - [ ] 数据切换时图表平滑过渡
  - [ ] 无数据时有友好提示

#### 功能8：目标追踪

**描述**：允许用户设定每周/每月运动目标，并追踪完成进度。

- **输入**：
  - 目标周期（每周/每月）
  - 目标次数（如每周运动4次）
  - 目标运动类型（可选，如"健身"或"任意运动"）

- **输出**：
  - 目标进度条（如"本周 3/4 次"）
  - 目标完成状态（进行中/已完成）
  - 目标完成时的庆祝提示

- **规则**：
  - 支持同时设置多个目标（如每周健身3次 + 每周篮球1次）
  - 目标周期结束时自动重置并计算完成率
  - 可修改或删除目标

- **验收标准**：
  - [ ] 支持设置至少3个并行目标
  - [ ] 进度条实时更新
  - [ ] 目标完成时有明显提示

#### 功能9：智能提醒

**描述**：基于用户最近的打卡记录，智能判断是否需要提醒。

- **规则**：
  - 如果距离上次打卡已经超过设定阈值（如2天），主动推送提醒
  - 提醒文案根据Streak状态变化（如"已经连续5天了，今天继续保持！"或"已经2天没运动了，动起来！"）
  - 智能提醒与固定时间提醒不重复发送

- **验收标准**：
  - [ ] 超期未打卡时触发提醒
  - [ ] 提醒文案个性化（基于Streak状态）
  - [ ] 不与固定提醒重复

#### 功能10：自定义运动类型

**描述**：允许用户添加预设列表中没有的运动类型。

- **输入**：
  - 运动类型名称（最多10个字）
  - 选择图标（从预设图标库中选择）

- **规则**：
  - 自定义类型与预设类型同等地位
  - 支持编辑和删除自定义类型（删除后历史记录仍保留显示）
  - 预设类型不可删除

- **验收标准**：
  - [ ] 支持添加至少10个自定义类型
  - [ ] 自定义类型在打卡界面正常显示和使用
  - [ ] 删除自定义类型不影响历史记录

## 界面设计

### 页面列表

#### 页面1：首页（Dashboard）
- **用途**：应用主页面，展示核心数据和快速打卡入口
- **核心元素**：
  - 顶部：当前Streak天数（大字号，火焰图标）
  - 中部：本周/本月运动次数统计卡片
  - 中部：快速打卡区域（运动类型图标网格，点击后确认打卡）
  - 底部：最近几次打卡记录预览（显示前3条）
  - 底部导航栏：首页 | 历史 | 成就 | 统计 | 设置
- **交互方式**：
  - 点击运动类型图标 → 弹出确认框（显示运动类型 + 当前时间 + 备注输入框）→ 确认后打卡
  - 下拉刷新：刷新统计数据
  - 点击Streak区域 → 进入Streak详情页

#### 页面2：历史记录页
- **用途**：查看和管理所有打卡记录
- **核心元素**：
  - 顶部：筛选栏（按运动类型筛选、按日期范围筛选）
  - 中部：时间线列表（按日期分组）
  - 每条记录：运动类型图标、名称、时间、备注、删除按钮
  - 底部：记录总数统计
- **交互方式**：
  - 点击筛选条件 → 列表即时过滤
  - 左滑/长按记录 → 显示删除选项
  - 点击记录 → 查看详情（可编辑备注和时间）

#### 页面3：成就页
- **用途**：展示成就徽章和解锁进度
- **核心元素**：
  - 顶部：已解锁成就数/总成就数
  - 中部：成就网格（已解锁彩色，未解锁灰色）
  - 每个成就：图标、名称、解锁条件、解锁时间（已解锁）/ 进度条（未解锁）
- **交互方式**：
  - 点击成就 → 查看成就详情和完成进度
  - 解锁新成就时自动弹出庆祝动画

#### 页面4：统计页
- **用途**：数据可视化展示
- **核心元素**：
  - 顶部：时间范围切换器（本周/本月/最近30天/全部）
  - 中部：柱状图（运动频率）
  - 中部：饼图（运动类型分布）
  - 底部：关键指标卡片（总次数、最爱运动、平均每周次数）
- **交互方式**：
  - 切换时间范围 → 图表数据更新
  - 点击图表元素 → 显示详细数据

#### 页面5：设置页
- **用途**：应用配置和个性化
- **核心元素**：
  - 提醒设置：固定时间提醒列表（添加/编辑/删除/开关）
  - 智能提醒：开启/关闭、超期阈值设置
  - 目标设置：添加/编辑/删除运动目标
  - 自定义运动类型：添加/编辑/删除
  - 数据管理：导出数据、清除所有数据
  - 关于：版本号、隐私说明
- **交互方式**：
  - 开关切换：即时生效
  - 时间选择：弹出时间选择器
  - 危险操作（清除数据）：二次确认

### 设计风格

- **整体风格**：极简、活力、现代
- **配色偏好**：
  - 主色调：活力橙/珊瑚色（#FF6B6B）——代表运动的热情和活力
  - 辅助色：深灰（#2D3436）——文字和图标
  - 背景色：纯白/浅灰（#F8F9FA）——清爽干净
  - 强调色：亮绿（#00B894）——成功、完成状态
- **字体**：系统默认字体栈，确保跨平台一致性
- **图标风格**：线性图标（Outline），简洁清晰
- **参考设计**：
  - 布局：类似Apple Health的卡片式布局
  - 动画：类似Duolingo的庆祝动画（成就解锁时）

## 非功能性需求

### 性能要求
- 首屏加载时间 ≤ 2秒（在3G网络下）
- 打卡操作响应时间 ≤ 100ms
- 历史记录加载100条 ≤ 1秒
- 图表渲染 ≤ 500ms

### 安全要求
- 数据本地存储（IndexedDB），不上传服务器
- 无需用户注册登录，无个人信息收集
- 清除数据功能需要二次确认，防止误操作

### 兼容性要求
- **必须支持**：Chrome（Android）、Safari（iOS）、Edge
- **PWA要求**：
  - 支持添加到主屏幕（Add to Home Screen）
  - 支持离线访问（Service Worker缓存核心资源）
  - 支持浏览器推送通知
- **响应式设计**：适配手机（375px-428px宽度）、平板、桌面
- **最低支持**：iOS 13+、Android 8+

### 可访问性
- 支持屏幕阅读器（ARIA标签）
- 颜色对比度符合WCAG AA标准
- 支持字体大小调整（使用相对单位rem）
- 所有交互元素支持键盘操作

## 限制条件和边界

### 明确不做的功能
1. **多用户/社交功能**：不支持注册登录、好友添加、分享、排行榜
2. **详细训练记录**：不记录健身动作、组数、重量、心率等详细数据
3. **GPS轨迹追踪**：不记录运动路线、速度、距离
4. **视频课程/训练计划**：不提供教学内容或跟练视频
5. **商城/电商**：不涉及任何商品销售
6. **云同步**：当前版本数据仅本地存储（P2可考虑导出备份）
7. **运动时长记录**：当前版本仅记录"是否运动"，不记录时长
8. **多语言支持**：当前版本仅支持中文

### 技术边界
- 纯前端应用，无后端服务器
- 数据存储在浏览器本地（IndexedDB），换设备/清浏览器数据会丢失
- 推送通知依赖浏览器权限，用户拒绝则无法使用提醒功能
- 离线模式下仅支持查看已有数据和本地打卡（PWA缓存）
